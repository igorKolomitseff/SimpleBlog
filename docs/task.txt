Тестовое задание для Junior Python/Django Backend-разработчика  


Задание 1: Простое Django-приложение (обязательное)  
Создайте блог с возможностью публикации статей.  

1. Модели:  
- Статья (Article)  
  - Поля:  
    - title (заголовок)  
    - content (текст статьи)  
    - author (связь с User, автоматически устанавливается при создании)  
    - created_at (дата создания, автоматически заполняется)  
    - is_published (статус: опубликована/черновик, по умолчанию False)  
- Тег (Tag)  
  - Поля:  
    - name (уникальное название)  
  - Связь Many-to-Many с Article  

2. Админка:  
- Настроить отображение статей (title, author, is_published).  
- Добавить фильтр по статусу (is_published).  
- Добавить действие "Опубликовать выбранные статьи".  

3. Представления и шаблоны:  
- Главная страница: список опубликованных статей (по 5 на страницу).  
- Страница статьи: полный текст, автор, дата создания.  
- Форма создания/редактирования статьи (только для авторизованных пользователей).

4. Тесты:  
- Проверить:  
  - Создание статьи.  
  - Фильтрацию опубликованных статей.  
  - Доступ к форме редактирования (только для автора).  

---

Задание 2: Простое API (обязательное)  
Создайте API для блога на DRF (Django REST Framework).  

1. Эндпоинты:  
- GET /api/articles/ – список статей.  
- POST /api/articles/ – создать статью (автор определяется автоматически).  
- GET /api/articles/<id>/ – детали статьи.  
- PUT/PATCH /api/articles/<id>/ – редактирование (только для автора).  
- DELETE /api/articles/<id>/ – удаление (только для автора).  

2. Дополнительно:  
- Добавить фильтрацию по статусу (?is_published=true).  
- Подключить простую документацию (Swagger или DRF Docs).  

---

Задание 3: Дополнительные задачи (по желанию)  
1. Оптимизация запросов 
   - Убедиться, что в списке статей нет N+1 проблемы (проверить через django-debug-toolbar).  
2. Кэширование
   - Закешировать главную страницу на 2 минуты.  
3. Аутентификация 
   - Добавить регистрацию пользователей (/api/auth/register/).
